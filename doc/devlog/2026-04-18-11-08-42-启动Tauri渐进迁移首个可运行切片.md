# 启动 Tauri 渐进迁移首个可运行切片

## 本次目标
- 按迁移设计文档落地首个可运行的 Tauri 切片。
- 补齐前后端骨架、真实配置链路、托盘/提醒基础能力与桌面构建验证。

## 核心改动
- 新增 React + TypeScript + Vite 前端壳层，提供设置页与提醒弹窗页面。
- 新增 `src-tauri/` Rust/Tauri 工程，接入托盘、提醒弹窗、后台调度、开机自启动与 Command 契约。
- 兼容现有 `%LocalAppData%/MiniNotifier/hydration-settings.json` 配置文件，保留原有提醒设置字段。
- 切换 Node 环境到 `24.15.0` 后重新安装依赖，完成前端和 Tauri 构建校验。

## 校验情况
- `node -v`：`v24.15.0`
- `npm run build`：通过
- `cargo check --manifest-path src-tauri/Cargo.toml`：通过
- `npx tauri build --debug`：通过，生成调试版 `msi` 与 `nsis` 安装包
- 应用级 smoke test：启动 `src-tauri\\target\\debug\\mininotifier-tauri.exe` 5 秒后确认进程存活，再主动结束

- 当前迁移切片已覆盖托盘、配置、提醒弹窗与基础调度，但尚未迁入原 WPF 版的全局鼠标活跃度判断。
- `src/MiniNotifier/` 旧 WPF 实现仍保留，当前属于并行迁移阶段，尚未切换默认入口。
- 目前使用的是兼容旧 JSON 的基础提醒文案服务，尚未完整移植 WPF 版那套更丰富的文案池。
