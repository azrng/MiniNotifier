# 自动与手动检测更新开发

## 本次目标

- 参考 ApixPress 的更新检测链路，为 MiniNotifier 接入启动自动检查更新与设置页、托盘手动检查更新。

## 核心改动

- 新增应用更新 DTO、更新检测服务和隐藏更新执行器。
- 新增 `appsettings.json` 更新源配置，并确保发布输出包含该配置文件。
- 设置窗口新增“软件更新”卡片，支持自动检查开关、手动检查、版本状态展示和立即更新。
- 托盘菜单新增“检查更新”入口，发现新版后先询问用户再启动更新。
- 本地设置 JSON 增加 `EnableUpdateCheck` 字段，默认开启。

## 校验情况

- `dotnet build MiniNotifier.slnx` 通过。
- `dotnet publish src\MiniNotifier\MiniNotifier.csproj -c Release` 通过。
- `dotnet run --project src\MiniNotifier\MiniNotifier.csproj -- --apply-update` 在缺少请求文件时按预期返回失败码，未进入普通窗口启动。
- 已确认发布目录包含 `appsettings.json`。

## 风险或遗留项

- 仓库当前没有测试项目，未补充 xUnit 单元测试。
- 更新包需要 GitHub Releases 后续实际发布 `versions.json` 和 zip 包后才能完成真实在线更新验证。
