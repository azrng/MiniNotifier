# 发布 1.0.3 版本

## 本次目标

发布包含更新检测、单实例启动、立即提醒时间重置和提醒弹窗卡顿优化的 `1.0.3` 版本。

## 核心改动

- 将应用版本号从 `1.0.2` 升级为 `1.0.3`。
- 在 GitHub Actions 打包流程中生成 `versions.json` 更新清单。
- 将 `versions.json` 作为 Release 附件上传，供应用内自动/手动检测更新使用。
- 本地按 Release 参数生成 x64、x86、framework-dependent 三类发布包并验证清单内容。

## 修改文件

- `src/MiniNotifier/MiniNotifier.csproj`
- `.github/workflows/build.yml`
- `TASK.md`

## 校验情况

- `dotnet build src\MiniNotifier\MiniNotifier.csproj --configuration Release`：通过，0 警告，0 错误。
- 本地发布包生成：通过，输出 `MiniNotifier-win-x64.zip`、`MiniNotifier-win-x86.zip`、`MiniNotifier-framework-dependent.zip`、`versions.json`。
- `dotnet test MiniNotifier.slnx --no-build`：已执行，当前解决方案未输出具体测试用例结果。
- `git diff --check`：通过，仅有 Git 换行提示。
- 发布版启动 smoke test：通过，`artifacts/local-release/publish/win-x64/MiniNotifier.exe` 可启动并保持运行。

## 风险或遗留项

- 远端 Release 需要推送到 `origin/main` 后由 GitHub Actions 实际生成，推送后需检查 Actions 与 Release 附件。
- 当前 `versions.json` 默认指向 x64 自包含包，应用内更新会优先下载 `MiniNotifier-win-x64.zip`。
