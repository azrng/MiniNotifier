# 2026-03-28 单文件发布补齐原生库内嵌

## 本次目标
- 解决已开启单文件发布后，发布目录仍然出现多份 WPF 原生 DLL 的问题

## 核心改动
- 在发布配置中补充 `IncludeNativeLibrariesForSelfExtract=true`
- 开启 `EnableCompressionInSingleFile=true`，减少单文件体积膨胀
- 将 `DebugType` 调整为 `embedded`，避免额外生成独立 `pdb`

## 修改文件
- `src/MiniNotifier/Properties/PublishProfiles/FolderProfile.pubxml`
- `TASK.md`

## 校验情况
- 执行 `dotnet publish .\src\MiniNotifier\MiniNotifier.csproj /p:PublishProfile=FolderProfile -c Release`，发布成功
- 检查发布目录，仅剩 `MiniNotifier.exe`

## 风险或遗留项
- 单文件可执行文件体积会明显变大，首次启动时会自解压原生库到临时目录，属于 .NET 单文件发布的正常行为
