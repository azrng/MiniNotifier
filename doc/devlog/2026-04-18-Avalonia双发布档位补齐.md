# 2026-04-18 Avalonia 双发布档位补齐

## 本次目标

- 为 Avalonia 主工程同时提供 `Native AOT` 和 `Single File` 两种正式发布档位。

## 核心改动

- 新增 `src/MiniNotifier.Avalonia/Properties/PublishProfiles/SingleFile-win-x64.pubxml`，固化 `win-x64` 单文件发布参数。
- 更新 `.github/workflows/build.yml`，让 CI 同时产出 `MiniNotifier-win-x64-aot.zip` 与 `MiniNotifier-win-x64-single-file.zip`。
- 在工作流中补充单文件发布目录校验，要求 `win-x64-single-file` 目录只包含一个 `MiniNotifier.Avalonia.exe`。
- 在 `src/MiniNotifier.Avalonia/MiniNotifier.Avalonia.csproj` 中新增发布后清理目标，移除发布目录中的 `.pdb` 文件，避免单文件档位残留额外符号文件。

## 风险或遗留项

- `Native AOT` 档位仍会保留第三方原生依赖文件，当前不追求物理单 exe。
- `Single File` 档位虽然目录只有一个 exe，但运行时仍会把原生库解压到临时目录，属于 .NET 官方单文件机制的正常行为。
